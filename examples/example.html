Core

How dot-grid Works

Every dot-grid layout uses three nested layers:

  • .grid-w — centered wrapper with configurable max-width (margin-inline: auto)
  • .grid-r — 12-column grid row. Also registers itself as a named container for its children
  • .grid-c-{n} — column spanning n of the 12 columns
<div class="grid-w">
  <div class="grid-r">
    <div class="grid-c-12 grid-c-md-6 grid-c-lg-4">Column</div>
    <div class="grid-c-12 grid-c-md-6 grid-c-lg-4">Column</div>
    <div class="grid-c-12 grid-c-md-6 grid-c-lg-4">Column</div>
  </div>
</div>

Why responsive classes work: .grid-r sets container-type: inline-size; container-name: grid. Responsive child classes like .grid-c-md-6 watch whether their own row is ≥768px wide — not the viewport. This is what makes components reusable in any layout context.

How Container Queries Work

Container queries let elements respond to their container's size, not the viewport. The same component stacks in a narrow sidebar and goes two-column in a wide main area — identical HTML, no JS.

In dot-grid: .grid-r is the named container (container-name: grid). .grid-c-md-6 fires when that row is ≥768px — regardless of how wide the browser window is.

Responsive Display Utilities

These fire when the parent .grid-r container reaches the breakpoint.

  • .hidden-{bp} — display: none at ≥bp
  • .block-{bp} — display: block at ≥bp
  • .flex-{bp} — display: flex at ≥bp
  • .grid-{bp} — display: grid at ≥bp

.hidden-md — visible below 768px, hidden at ≥768px

Usage reference

<!-- Hide on large containers, show on small -->
<div class="grid-c-12 hidden-lg">Mobile-only content</div>

<!-- Become block at md+ -->
<div class="grid-c-12 block-md" style="display: none">
  Shown at md+
</div>

<!-- Become flex at lg+ -->
<div class="grid-c-12 flex-lg">...</div>

<!-- Become grid at xl+ -->
<div class="grid-c-12 grid-xl">...</div>

When to Use: Container vs Viewport Queries

Container Queries grid-c-{bp}-{n}

  • Reusable components
  • Card grids
  • Widgets in different contexts
  • Component libraries
  • Most modern layouts

Viewport Queries

  • Top-level page structure
  • Navigation visibility
  • Hiding/showing elements globally
  • Typography scaling with viewport
Advanced & Config

Subgrid

.grid-r-subgrid makes a grid item inherit its parent's column tracks (grid-template-columns: subgrid). Use it to align nested content across the parent grid.

<div class="grid-r">
  <div class="grid-r-subgrid">  <!-- spans 1/-1, inherits parent columns -->
    <div class="grid-c-4">Nested 1</div>
    <div class="grid-c-4">Nested 2</div>
    <div class="grid-c-4">Nested 3</div>
  </div>
</div>

Nested 1 (grid-c-4)

Nested 2 (grid-c-4)

Nested 3 (grid-c-4)

.grid-c-subgrid is the row counterpart: it inherits the parent's row tracks (grid-template-rows: subgrid; grid-row: 1 / -1). Use it for equal-height card content alignment across a row.

Nested Rows

A .grid-r placed as a direct child of another .grid-r automatically spans all 12 columns (grid-column: 1 / -1) and removes its bottom padding to avoid double-spacing.

Overridable variables

Variable Default Controls
--grid-columns 12 Number of columns in a row
--grid-gap clamp(1rem, 2vw, 2rem) Gap between columns and rows
--grid-w-max-width 190rem Max width of .grid-w
--grid-w-width 90vw Preferred width of .grid-w
--grid-w-min-width 36vw Min width of .grid-w

Note on --grid-bp-* variables: dot-grid exposes --grid-bp-sm, --grid-bp-md, etc. as :root variables for documentation purposes, but they are read-only metadata. CSS custom properties cannot be evaluated inside @container or @media conditions — @media (min-width: var(--grid-bp-md)) does not work. Use the literal pixel values (640px, 768px, 1024px, 1280px, 1536px) in your own queries.
